<definition>
Definition
attributes
code: key required
Identificador único de la definición
Preguntar por el formato de este codigo. Sugerencias: ruta del fichero separada por puntos. Ejemplo: micv.definitions.
parent: Code optional
Indica la relación de herencia de la definición. Se especifica el código de la definición de la que se heredan los atributos, propiedades y declaraciones
{ ONLY Definition FROM Model }
properties
abstract optional
Su presencia indica que la definición es abstracta
Antes era un atributo
singleton optional
Su presencia indica que la definición representa un tipo de entidad de la que sólo puede existir una instancia
Antes era un atributo
environment optional
Su presencia indica que la definición representa un tipo de entidad que se puede utilizar para configurar el entorno del usuario en el espacio de negocio
Antes era un atributo
label multiple optional
Etiqueta descriptiva de una definición. Se utiliza para ofrecer a los usuarios del espacio de negocio un significado sobre un tipo de un entidad
language: (es,en,de,fr,it,pt) required
Idioma en el que se expresa la etiqueta
type: (short,long) = short optional
Tipo de etiqueta: corta o larga. En función del espacio en la interfaz de usuario, se puede mostrar al usuario una etiqueta corta o larga. En caso de no existir la descripción larga, se toma la descripción corta
content: text
description multiple optional
Descripción explicativa de una definición. Se utiliza para ofrecer a los usuarios una explicación más detallada sobre un tipo de entidad
language: (es,en,de,fr,it,pt) required
Idioma en el que se expresa la descripción
content: text
help optional
Página de ayuda asociada a la definición. El contenido debe estar en formato HTML
language: (es,en,de,fr,it,pt) required
Idioma en el que se ha expresado el fichero de ayuda
filename: String required
Fichero HTML que contiene la ayuda. Este fichero se ubica en el directorio "/help/(lang)" del modelo de negocio
tag multiple optional
Etiqueta para facilitar la busqueda de definiciones

FieldDeclaration
Descripción del campo de un formulario
attributes
code: key required
Identificador único de la declaración del campo
name: string optional
La presencia de este atributo indica que el contenido del campo es accesible para realizar cálculos u operaciones. Para acceder al campo se utiliza name como índice en la lista de campos de un formulario o una sección
Hablar con Mario para valorar cómo se hace en BPI
properties
required optional
La presencia de esta propiedad indica que es obligatorio cumplimentar el campo. Actualmente no admite expresiones, pero en el futuro se permitirá definir expresiones.
multiple optional
La presencia de esta propiedad indica que es un campo replicable en el formulario. Cuando un campo es replicable, en el espacio de negocio el usuario puede añadir instancias del campo, por ejemplo para introducir varios teléfonos en una ficha de datos personales
default optional
Contiene una constante o una expresión con la se inicializa el valor de un campo cuando se crea el formulario
content: xml
label optional
Contiene una etiqueta descriptiva del campo. Se utiliza para ofrecer a los usuarios del espacio de negocio un significado del campo declarado
language: (es,en,de,fr,it,pt) required
Idioma en el que se expresa la etiqueta
type: (short,long) = short optional
Tipo de etiqueta: corta o larga. En función del espacio en la interfaz de usuario, se puede mostrar al usuario una etiqueta corta o larga. En caso de no existir la descripción larga, se toma la descripción corta
content: text
description optional
Contiene una descripción explicativa del campo. Se utiliza para ofrecer a los usuarios una explicación más detallada del campo declarado
language: (es,en,de,fr,it,pt) required
Idioma en el que se expresa la descripción
content: text
help optional
Referencia a una página de ayuda del campo. Se utiliza para mostrar una página de ayuda del campo declarado. El contenido debe estar en formato HTML
language: (es,en,de,fr,it,pt) required
Idioma en el que se ha expresado el fichero de ayuda
filename: String optional
Fichero HTML que contiene la ayuda. Este fichero se ubica en el directorio "/help/(lang)" del modelo de negocio
